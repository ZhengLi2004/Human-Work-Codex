# Skill: Work — Post-Report Assessment

## 目标

在人类基于结果包完成实验报告后，结合人类解释、完整证据和整体研究路线，判断继续开展同类实验是否具有足够信息增益，并提出下一 TODO 建议。Work 可以建议拒绝机械堆叠实验，最终决定交给人类进行。

## 输入

- 人类实验报告；
- 人类原始研究报告；
- 当前 TODO；
- Codex 执行包；
- 完整聚合表、论文候选表和列变换清单；
- 相关相邻 TODO 和已有研究证据。

## 评估顺序

### 1. 以人类报告为科学解释来源

提取人类已经总结的：

- 主要观察；
- 规律与机制解释；
- 适用条件和限制；
- 尚未解决的问题；
- 人类明确关心的下一方向。

Work 不重新取代人类写机制结论。

### 2. 判断继续实验的信息价值

评估：

- 当前证据是否已足以支撑研究目的；
- 证据不足是否能由有限新实验明确解决；
- 继续增加样本、参数或场景是否只会重复已有规律；
- 噪声、指标分辨率、实现限制或研究设计是否使继续堆叠收效甚微；
- 新实验是否能区分至少两个仍有竞争力的解释；
- 预计资源与信息增益是否匹配；
- 是否应先修改方法、指标或问题，而不是继续相同实验。

### 3. 形成建议

建议属于以下之一：

- `continue_same_research_line`；
- `targeted_followup`；
- `redesign_before_more_experiments`；
- `stop_evidence_sufficient`；
- `stop_low_expected_information_gain`；
- `propose_new_phenomenon_experiment`。

### 4. 候选新现象

当建议停止原实验时，可以从论文候选表中提出值得独立研究的候选现象，但必须：

- 引用具体任务变量组合或表格行；
- 明确它只是观察规律，不是已确认机制；
- 说明最小什么新实验能够区分可能解释；
- 与原 TODO 边界分离；
- 等待人类批准后才创建正式 TODO。

## 输出

- 更新 TODO 第 5 节；
- 简短的信息增益评估；
- 必要时的候选下一 TODO 标题、核心问题和依据；
- workflow 状态 `awaiting_human_next_todo_decision`。

人类批准后，才由 `research-report-to-todos` 创建下一正式 TODO。
