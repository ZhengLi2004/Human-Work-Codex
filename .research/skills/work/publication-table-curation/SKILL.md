# Skill: Work — Publication Table Curation

## 目标

在 Codex 完成整个 TODO 并返回执行包后，把完整聚合表整理成可直接交给人类观察和撰写论文或实验报告的候选表。Work 只做列级与展示级整理，不改变行数和实验事实。

## 启动条件

- workflow 为 `codex_completed`；
- 存在 `CODEX_EXECUTION_BUNDLE.md`；
- TODO 已引用全部阶段证据；
- 每个实验表都有 `runs`、`aggregate_full` 和 manifest；
- Codex 已完成完整组合与重复校验。

尚未得到 Codex 返回内容时，Work 不得进入本 Skill。

## 数据质量复核

先检查：

- `aggregate_full` 行数等于计划组合数；
- 每个组合有 `n_planned`、`n_valid`、离散性和覆盖状态；
- 失败、缺失、不可比和重复不足组合仍在表中；
- 没有跨控制条件平均；
- 每行能追溯到 `runs` 与原始数据；
- TODO 中的引用路径存在。

发现问题时将收尾流程标记为 `blocked`，向人类报告需要重新调用 Codex 处理的具体问题 Codex。Work 不得自行重算、补数或删行修复。

## 不可违反的行约束

- 每个输入 `aggregate_full` 单独生成一个输出 `publication_table`；
- `output_rows == input_rows`；
- 不删除、筛选、复制、合并、去重或重新聚合任何行；
- 不只保留最好组合；
- 不改变任务变量组合含义；
- 不隐藏失败、缺失、负结果或覆盖状态。

## 允许的列操作

- 删除纯操作性列，如原始路径、内部 run ID、缓存键、调试字段；
- 调整列顺序；
- 将内部变量名改为论文可读名称；
- 将枚举值映射为可读标签；
- 合并一一可追溯的展示列；
- 选择论文需要的指标列，但保留影响解释的覆盖、失败和关键指标；
- 做有记录的显示舍入；
- 按任务变量确定性排序。

## 列变换清单

每张输出表必须记录：

- 输入和输出路径、哈希；
- 输入和输出行数；
- 稳定组合键；
- 删除列；
- 列重命名；
- 枚举映射；
- 展示列合并规则；
- 指标选择与舍入；
- 排序键；
- 行数相等校验；
- 输出行到输入行的追踪方式。

## 输出

- 一个或多个 `publication_table`；
- 对应列变换清单；
- `RESULT_PACKAGE.md`；
- TODO 第 5 节的文件引用；
- 提交给人类的结果包。
