# Skill: Batch Execution

## 目标

为多样本、多任务变量组合或多重复实验提供可恢复、可观察、资源受控且不会遗漏计划单元的执行器。

## 任务清单

运行前必须落盘完整任务清单。最小可恢复单元由以下内容唯一确定：

- 任务变量组合；
- 重复标识；
- 数据分片或样本范围；
- 代码、数据和配置版本。

不得边看结果边选择是否生成后续任务。

## Checkpoint 与幂等

- 同一命令可直接续跑；
- 只有产物完整性校验通过的单元才可跳过；
- 状态写入尽可能原子化；
- 中断不得产生重复计数或静默遗漏；
- 并发不得破坏共享状态；
- 重跑使用新运行标识，不覆盖历史原始数据；
- 提供一个小规模中断—续跑测试证明恢复正确。

## 进度与日志

- 进度粒度与可恢复任务单元一致；
- 展示已完成数、总数、失败数、当前任务和 ETA；
- 交互终端维护黄色高亮实时进度；
- 非交互环境降级为普通日志；
- 终端与文件双写；
- 记录变量、重复、开始结束时间、状态、错误和产物路径。

## 资源

- 默认不超过任何单项可用资源的 50%；
- 并发数根据实测单任务 CPU、内存和显存峰值确定；
- 优化不得改变样本集合、随机过程、指标定义或比较公平性；
- 预计超过 30 分钟时，Codex 只完成测试和预检，直接向人类交付一行命令并阻塞等待，不返回 Work。

## 完成检查

批量执行结束后比较：

- 计划任务单元数；
- 成功、失败、缺失和重复单元数；
- checkpoint 状态；
- 原始产物可读性；
- 与 `aggregate_full` 的组合和重复覆盖；
- 是否存在重复计入或未声明过滤。
