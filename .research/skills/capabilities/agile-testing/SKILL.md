# Skill: Agile Research Testing

## 目标

基于软件工程中的风险驱动测试、测试金字塔、左移测试、持续反馈和回归保护，为科研代码提供最小充分但可信的验证。测试服务于科学正确性，避免不必要的流程负担。

## 1. 先定义风险，而不是先枚举测试类型

每个实现切片开始前，写出最可能导致错误科研判断的 1-3 个风险：

- 数学或算法语义错误；
- 单位、尺度、维度、索引、标签或数据泄漏；
- 对照组除目标因素外发生变化；
- 指标实现或聚合错误；
- 随机、缓存、并发或 checkpoint 改变样本集合；
- 性能优化改变数值或实验语义；
- 结果保存、恢复或表生成遗漏组合。

然后选择能够最快区分这些风险的测试。不是每个任务都需要单元、集成、系统和回归四套完整仪式。

## 2. 测试金字塔

### 快速基础层

适合频繁运行：

- 微型人工真值样例；
- 纯函数单元测试；
- 参数化边界测试；
- 不变量、性质、变形和差分测试；
- 数据 schema、单位和维度契约。

### 中间集成层

只覆盖关键边界：

- 数据加载到方法输入；
- 方法输出到指标；
- 指标到结果保存；
- checkpoint 到恢复；
- 对照配置到实际运行参数。

### 顶层端到端

数量少但必须真实：

- 一个任务变量组合；
- 一次重复；
- 最小代表数据；
- 完整的数据-方法-指标-保存-聚合链路。

不得把完整 Pilot 或 Confirmation 当作日常测试套件。

## 3. 软件工程测试技术在科研中的选用

| 风险类型 | 优先测试技术 | 敏捷落地方式 |
|---|---|---|
| 公式、变换、局部算法 | 单元、性质、差分、变形测试 | 使用小型确定输入，秒级运行 |
| 数据语义、单位、形状 | 契约与参数化测试 | 在加载边界立即断言 |
| 旧实现缺乏测试 | 特征测试 | 先固定当前关键行为，再局部修改 |
| 模块衔接 | 集成测试 | 只覆盖关键接口，不复制全系统 |
| 完整实验链 | 端到端冒烟测试 | 一个组合、一次重复、最小数据 |
| 缺陷修复 | 回归测试 | 先复现缺陷，再固定最小用例 |
| 并发与恢复 | 幂等、故障注入、恢复测试 | 人为中断一个小任务并续跑 |
| 性能与资源 | 重复 benchmark | 同条件多次测量，报告离散性 |

## 4. 阶段化测试策略

### Exploration

- 使用探针、断言、数据剖面和微型样例；
- 目标是快速证伪错误前提；
- 一次性探针无需立即工程化；
- 一旦探针保护了关键不变量或发现真实缺陷，应转为自动化测试。

### Development

- 对核心算法建立快速单元与性质测试；
- 对项目边界建立契约和集成测试；
- 每个可工作的纵向切片完成后立即运行相关测试；
- 修复缺陷时添加回归测试；
- 阶段结束前完成一个端到端冒烟测试。

### Pilot

- 先验证一个组合、一次重复的完整链路；
- 验证参数枚举、重复生成、指标、资源记录和聚合；
- 对参数脚本进行小规模恢复测试；
- Pilot 数据不是测试替代品；测试失败时先修复，不继续扩大网格。

### Confirmation

- 运行冻结规格相关的关键回归套件；
- 验证正式配置、任务清单、checkpoint、日志、进度和聚合；
- 完成一个组合、一次重复预检；
- 正式运行脚本不得在预检失败时交付人类。

### Ablation / Diagnosis

- 验证干预只改变目标因素；
- 检查对照配置和指标与主实验兼容；
- 使用最小对照区分候选解释；
- 诊断脚本发现的关键故障应形成回归测试。

## 5. 快速反馈节奏

- 编辑循环：运行受影响的秒级测试；
- 完成纵向切片：运行该切片的快速套件与必要集成测试；
- 阶段审查前：运行当前阶段最小充分套件；
- 长时运行前：运行正式入口预检与恢复检查；
- 合并关键修改前：运行相关回归套件。

测试选择应与变更表面相关，不要求每次运行整个项目测试。

## 6. Oracle 与数值稳定性

- 有解析答案时使用解析答案；
- 有可信参考实现时使用差分测试；
- 没有精确答案时使用不变量、单调性、对称性、守恒关系或变形关系；
- 容差必须由浮点精度、算法误差或统计性质支持；
- 随机逻辑测试固定种子，统计结论独立重复；
- 避免仅检查非空、非 NaN 或退出码；应检查科学语义。

## 7. 最小完成定义

一个实现切片可以继续，需要：

- 当前最高风险有对应测试或明确说明为何无法自动测试；
- 相关快速测试通过；
- 失败信息能够定位到研究对象或接口；
- 没有以扩大实验规模掩盖低层错误。

一个阶段可以提交人类审查，需要：

- 关键回归、集成和冒烟测试均满足当前阶段要求；
- 所有失败和未测风险被明确报告；
- 测试产物已写入 TODO 和阶段审查包。
