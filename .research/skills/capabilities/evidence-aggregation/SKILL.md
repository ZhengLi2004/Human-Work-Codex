# Skill: Evidence Aggregation

## 目标

把实验原始数据转化为完整、可追溯、不可选择性删减的运行表和组合聚合表。任何向人类报告的实验指标，都必须基于相同任务变量组合、相同控制条件下的多次独立重复。

## 聚合前必须存在

- 任务变量及全部计划取值；
- 控制变量及固定值或规则；
- 完整计划组合集合；
- 独立重复单位和每组合重复数；
- 指标定义、方向和来源；
- 预先定义的失败、缺失和排除规则；
- 原始数据目录与版本标识。

缺少这些定义时，不得根据已经看到的结果反推分组规则。

## 输出 1：`runs`

每个独立运行一行，至少包含：

- 稳定 run 标识；
- 全部任务变量；
- 控制配置或其可追溯哈希；
- 重复标识；
- 运行状态；
- 每个指标的单次运行值；
- 原始文件引用；
- 失败、排除和异常原因；
- 代码、数据与配置追踪信息。

单次运行值仅用于追踪，不可作为 TODO 或阶段审查中的代表性研究指标。

## 输出 2：`aggregate_full`

每个计划任务变量组合恰好一行。聚合键包含全部任务变量；若控制变量发生变化，则加入分组键或标记不可比。

默认统计单位是独立重复：先在每个重复内计算指标，再在同一组合的重复之间汇总。不得把多个重复的底层样本直接拼接后虚增独立样本量。配对设计应先在配对内计算差值或效应，再跨配对重复汇总。只有人类报告或预先批准的实验规格明确规定池化估计量时，才可采用其他聚合层级，并必须在 manifest 中说明。

每个指标至少输出：

- `mean`；
- `std`、置信区间或人类批准的其他离散性量度；
- `n_planned`；
- `n_valid`；
- `coverage_status`；
- 必要时的失败数与缺失数。

必须保留：

- 完整成功组合；
- 重复不足组合；
- 全部失败组合；
- 未运行或缺失组合；
- 不可比较组合。

缺失指标使用空值和状态字段，不得删除整行。

## 输出 3：`aggregation_manifest`

至少记录：

- 实验规格路径和版本；
- 原始输入索引；
- 分组键；
- 指标公式或计算代码标识；
- 重复要求；
- 失败、缺失、排除和异常明细；
- 计划组合数、输出行数和覆盖率；
- `runs` 与 `aggregate_full` 路径；
- 文件哈希或等价追踪信息；
- 聚合时间和工具版本。

## 自动校验

聚合后必须检查：

- `aggregate_full` 行数等于计划组合数；
- 每个组合的 `n_planned`、`n_valid` 与 `runs` 一致；
- 不存在跨控制条件平均；
- 没有未声明的行过滤；
- 同一原始运行未重复计入；
- 每个聚合行可追溯到来源运行；
- 重新聚合不会覆盖原始数据；
- TODO 引用路径存在且可读取。

## 及时 TODO 登记

每个实验性阶段完成后，Codex 在提交人类审查前必须：

1. 完成上述聚合和校验；
2. 在 TODO 追加 E-xx；
3. 引用原始数据、`runs`、`aggregate_full` 和 manifest；
4. 报告全部组合覆盖、均值、离散性、失败和异常；
5. 标记不完整或不可比证据；
6. 记录及时报告时间。
