# Skill: Codex — Confirmation

## 阶段目标

在经人类冻结的任务变量、控制条件、参数、指标和重复策略下，准备并验证正式实验入口；人类完成长时运行后，Codex继续核验全部结果并完成完整聚合。Confirmation 不承担继续调参。

## 正式运行前

- 生成冻结的实验规格；
- 验证完整任务变量组合无遗漏；
- 验证每组合重复数和独立重复生成规则；
- 运行与冻结代码相关的关键回归测试；
- 验证数据—方法—指标—保存—聚合链路；
- 验证 checkpoint、幂等恢复、日志和进度；
- 完成一个组合、一次重复的正式配置预检；
- 将预检与正式结果清晰区分。

## 一行命令与人类运行

预计超过 30 分钟时，Codex必须直接向人类提交：

- 一行可从约定目录运行的命令；
- 预计时间、CPU、内存、GPU/显存和并发；
- checkpoint 与恢复方式；
- 日志、原始数据和聚合输出路径；
- 完成判据和失败判据。

Codex 随后在同一执行调用中阻塞等待。人类运行结果可用后，Codex 原地恢复。

## 结果返回后

- 检查全部计划组合和重复；
- 检查控制变量一致性；
- 保留失败、缺失、重复不足和不可比组合；
- 生成 `runs`、`aggregate_full` 和 manifest；
- 向 TODO 追加 E-xx；
- 报告均值、离散性、覆盖和异常；
- 不根据结果重新调参、改指标或改变排除规则；
- 不替人类撰写机制结论。

## 阶段输出

- 冻结实验规格；
- 经过预检的一行正式命令；
- 可恢复正式运行脚本；
- 不可变原始数据；
- `runs`、`aggregate_full` 和 manifest；
- TODO 的 T-xx、E-xx；
- 直接提交人类的阶段审查包和 Codex 初始建议。

阶段结束后直接等待人类决定。只有人类批准完成整个 TODO 时，Codex 才生成执行包并返回 Work。
